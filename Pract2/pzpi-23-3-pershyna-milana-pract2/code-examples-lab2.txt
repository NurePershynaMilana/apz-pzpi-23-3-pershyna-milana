ВИКОРИСТАННЯ ШІ ПІД ЧАС РОЗРОБКИ
Під час виконання практичної роботи було використано інструмент штучного інтелекту Claude:
назва інструменту та версія: Claude (модель Claude Sonnet 4.6, план Free);
мета використання: допомога у поглибленні розуміння архітектурних рішень Figma та генерація пояснень до технічних концепцій презентації;
внесок автора: отримані результати були перевірені, проаналізовані та відредаговані, після чого адаптовані відповідно до теми архітектури Figma та вимог до оформлення презентації.

Запит (prompt) для пояснення алгоритму Operational Transformation:
Поясни простими словами як працює алгоритм Operational Transformation. Чому саме він використовується для синхронізації в реальному часі, а не просто перезапис даних? Відповідь формальна, для слайду презентації.
Отримана відповідь:
Claude пояснив принцип роботи OT-алгоритму через покроковий сценарій: два користувачі одночасно вносять зміни, сервер отримує обидві операції, трансформує їх відносно одна одної і розсилає узгоджений результат усім учасникам. Обґрунтовано перевагу OT над простим перезаписом — збереження змін усіх учасників без втрати даних. Пояснення використано для формування слайду 8 презентації.

Запит (prompt) для порівняння Figma з конкурентами:
Порівняй архітектурні підходи Figma, Sketch, Penpot та Framer. Що є ключовою перевагою Figma з технічної точки зору? Відповідь коротка, формальна, для слайду презентації.
Отримана відповідь:
Claude сформував чіткі формулювання для кожного порівняння: Sketch є нативним macOS-застосунком з локальним зберіганням, тоді як Figma перевершує його хмарною співпрацею; Penpot виграє у контролі даних, але поступається функціональністю; Framer орієнтований на прототипи з кодом, тоді як Figma ефективніша для командної роботи. Формулювання були використані на слайді 10 презентації.
